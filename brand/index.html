Brand system

mire

A retained-mode TUI runtime for F#. The brand is a box-drawing mark, a terminal mono identity, a readable documentation face, and one emerald moment.

Mark

└ is the framing primitive Mire renders with. It is not a decorative icon.

Type

Color

Overview

Use the manual as the source of brand behavior.

This page documents the reusable decisions: type roles, color roles, logo usage, badges, terminal surfaces, voice rules, and violations.

Core formula

└ + JetBrains Mono + Emerald #1A9A7E

That is the identity. Web documentation may add IBM Plex Sans for reading, but the brand signal stays mono.

Hierarchy rule

Size, weight, position, neutral contrast, then color.

Color is not the default way to create emphasis. Use it last, and use it once.

Typography

Two faces, two jobs.

IBM Plex Sans handles reading. JetBrains Mono preserves the terminal identity and renders code, labels, tokens, badges, navigation, and specimens.

Reading rules

  • Body text uses line-height: 1.55 and a max width near 62ch.
  • Headings and labels use letter-spacing: 0. Do not tighten mono text.
  • Use weights 400, 500, 600, and 700. Never use 300.

Mark

The mark is a box-drawing corner.

The symbol is └, the framing primitive Mire renders with. It is not a literal M, badge, mascot, or app icon shape.

Do

  • Use symbol-mono.svg when the context sets the color.
  • Use symbol.svg for the single accent moment on a surface.
  • Preserve the 2px safe padding and the original aspect ratio.

Do not

  • Frame it in a circle, pill, or rounded square.
  • Add shadow, glow, outline, gradient, rotation, or animation.
  • Place it on a busy background.

Voice

Write like a tool manual, not a launch page.

Lead with behavior, show real output, name limits plainly, and remove adjectives that do not carry facts.

01

Lead with the verb.

What it does comes first. The category can come later.

Diffs a cell grid and paints only the runs that changed.
Mire is a framework that provides a rendering layer.
02

Numbers or nothing.

"Fast" needs a number or it gets cut.

Redraws at ~30 FPS; a diff touches only changed cells.
Blazing-fast, buttery-smooth rendering.
03

You, never "users".

Address the reader directly.

You write update and view; Mire owns the loop.
Users implement their update logic.
04

Show, do not say.

A terminal frame beats a paragraph. Real code beats a feature list.

┌ counter ──────────┐
│ count: 7          │
│ [+] inc  [-] dec  │
└───────────────────┘

Rules

The brand breaks when decoration replaces structure.

Shipping any of these in a Mire surface means the brand has drifted.

Show violations
Gradients

Purple/blue gradients, mesh backgrounds, or any gradient at all.

Glass

Blurred cards, translucent panels, and floating glow effects.

Bento

2x2 marketing grids or "Fast / Simple / Powerful" feature cards.

Logo walls

Trusted-by strips, icon grids, and generic social proof.

AI badges

"Powered by AI" framing or product-copy shortcuts.

Emoji

No emoji in copy, headings, commits, or release notes.

Light weights

Font weight 300 reads as low-contrast slop.

Generic claims

"Build the future" and "all-in-one platform" do not belong here.
