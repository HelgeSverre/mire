# Changelog

All notable changes to the Mire project are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## [Unreleased]

### Changed

- **`Program.mkProgram` is now `Program.create`** — a clearer, action-first name that reads `Program.create init update view`. `Program.mkProgram` stays as a source-compatible alias, so existing code keeps working.

### Fixed

- **Selected table-row highlight now spans colored columns.** A `Table.view` selected row drew each column in its own style, so a colored cell punched a hole in the highlight (and a column foreground matching the inverse-video selection background vanished). The selected row is now drawn uniformly in the selection style via the new `Backdrop.recolor`.

## [0.5.0] - 2026-06-18

The agent layer plus a round of framework completeness: text selection & soft-wrap,
true grapheme clusters, Kitty graphics + `ImagePreview`, light/dark theme
notifications, runtime mouse hit-testing → focus, and a `Mire.Agent` layer
(`ChatTranscript`/`PromptBox`/`ApprovalModal`/`DiffView`) with a `samples/AgentShell`
MVP and a `samples/Gallery` showcase. A full user guide lives in `docs/guide/`.

### Changed

- **Agent demo dogfoods the new APIs.** `Mire.Demo.Agent` now routes its @mention / /slash completion through `PromptBox.completionToken`/`acceptCompletion` (dropping its hand-rolled token detection) and resolves the approval modal's Accept/Deny clicks through the runtime's region table via `Program.withMouseRegion` + `ApprovalModal.acceptRegion`/`denyRegion` (retiring the hand-mirrored `buttonHit` geometry; `buttonHit` remains as a utility). Verified end-to-end via `--dump` and pty.

### Added

- **Widget gallery sample (`samples/Gallery`).** A pure-framework showcase (references only `Mire`) exercising every base widget in its key states across 7 tabbed pages — Text/type/badges/key-hints, Boxes/panels/separators/split/highlight, Inputs (incl. selection + soft-wrap), Lists/Tables, Controls (toggles/progress/spinner/status bar), Overlays (modal/completion), and Media (ImagePreview/Markdown) — on the default brand theme. `just gallery`; `--dump` renders every page headlessly.
- **Mouse hit-testing → focus (runtime-owned region table).** A new `LayoutNode.Focusable` node (`Widgets.Focusable.region id child`) tags a subtree with a `RegionId`. The runtime retains `Layout.collectRegions` of each rendered frame and hit-tests mouse events through `Layout.regionAt`, delivering the topmost hit `RegionId` (or `None`) to a new `Program.withMouseRegion` handler — so an app can focus/activate by id instead of hand-computing rects. Regions nested inside a `Scroll` are omitted (their rects are in virtual content space). Default behavior is unchanged (mouse still flows to `MapInput`).
- **`ChatTranscript` virtualization + follow-tail.** `ChatTranscript.view` is the live transcript view: a scroll region that builds **only the blocks intersecting the viewport** into the `Scroll` node (the `Scroll` primitive otherwise renders the whole transcript onto an off-screen surface every frame) while a `ScrollView.scrollbar` (now public) reflects the true content. Adds `contentHeight`/`blockHeights`/`toBottom`/`clampOffset`/`atBottom` scroll-math helpers so apps stop re-deriving them; the offset stays app-owned MVU state. The Agent demo renders its transcript through `ChatTranscript.view`.
- **`PromptBox` completion + history.** `Mire.Agent.PromptBox` now carries a submit-`History` ring (`submit` pushes and clears, skipping blanks/dups; `historyPrev`/`historyNext` walk it with draft-preserving up/down recall) and locates the completion token under the caret (`completionToken triggers` → the `/`-or-`@` token + query + start; `acceptCompletion` splices a chosen value with a trailing space). The candidate source (commands, files) stays app-owned; any live edit exits history browsing.
- **`ImagePreview` widget + Kitty graphics protocol.** `Widgets.ImagePreview.render` draws a portable text fallback — a bordered, captioned box with pixel dimensions — that lands in the cell grid on every terminal. On Kitty/Ghostty an app overlays the real image with `Cmd.kittyImage col row cols rows pngBase64` (transmit-and-display, payload chunked at 4096 base64 bytes per the protocol). New general `Cmd.writeRaw` (the out-of-band escape hatch the clipboard and graphics commands are now built on), `Cmd.clearImages`, and `ANSI.kittyImage`/`ANSI.deleteImages`. The framework never decodes images. _Note:_ `Cmd.dispatch`'s second hook is now a general `writeRaw: string -> unit` (was `setClipboard`).
- **True grapheme clusters.** `Mire.Core.Grapheme` now segments text into UAX #29 grapheme clusters (`clusters`) and measures width by code point (`scalarWidth`/`clusterWidth`/`isWideScalar`/`isZeroWidthScalar`): astral scalars (surrogate pairs), emoji-ZWJ sequences, regional-indicator flags, and VS15/VS16 presentation selectors all resolve to the right column count, and `stringWidth` sums clusters (with an ASCII fast path). `Surface.Write`/`WriteClipped` iterate clusters, so an astral glyph or emoji cluster lands in a single cell instead of split surrogate halves. Zero-width detection uses Unicode general categories (exact); wide ranges approximate `EastAsianWidth.txt`.
- **Light/dark theme notifications (DEC mode 2031).** `Program.withThemeNotifications true` enables the terminal's color-scheme reporting (Contour/Kitty/Ghostty) and queries the current scheme at startup; `InputParser` decodes `CSI ? 997 ; 1|2 n` into a new `ThemeChanged Dark`/`ThemeChanged Light` `InputEvent` (new `Mire.Core.ColorScheme`), delivered through `MapInput` like any other input so an app can retheme on the fly. New `ANSI.enableThemeNotifications`/`disableThemeNotifications`/`queryColorScheme`.
- **Kitty private-use functional key decoding.** `InputParser` now maps the Kitty keyboard protocol's PUA functional codepoints (57344–57454) — numeric keypad (→ arrows/nav/digits/operators), F13–F35 (→ `Function`), and media/lock/lone-modifier keys (→ `Unknown`) — instead of letting them fall through as a meaningless PUA glyph insert. Input decoding is now feature-complete for the targeted terminals.
- **Text selection across the editing stack.** `Mire.Core.TextBuffer` gains an `Anchor: int option` and selection ops (`selection`/`hasSelection`/`selectAll`/`selectWord`/`extend`/`deleteSelection`); inserts/deletes replace the selection. `Mire.Core.TextEdit` adds `SelectAll`/`Select <motion>` actions and binds shift+arrows/Home/End to extend a selection and Ctrl/Cmd+A to select-all. `Widgets.Input.render` and `Widgets.TextArea.render` paint the selected range with the cursor style.
- **`TextArea` soft-wrap.** `Widgets.TextArea.wrapLine` word-wraps a logical line into visual segments (breaking at spaces, hard-breaking over-long words; partitions the line exactly so offsets map back to the buffer), and `Widgets.TextArea.renderWrapped` renders/scrolls/selects over those visual rows — a soft-wrapping sibling of `render` (apps pick wrap vs. horizontal-scroll per call site).
- **`Mire.Agent` — the agent-domain widget layer (0.5.0, in progress).** A new class library above `Mire.App` that references `Mire` only (one-directional chain; the base framework still knows nothing of agent concepts). First widget extracted: **`ChatTranscript`** + the **`TranscriptBlock`** model (user/assistant/thinking/tool-call/diff/table/error/notice/file-tree/task-timeline/plan blocks), lifted from `Mire.Demo.Agent`'s `Blocks.fs` and parameterized by `Mire.Widgets.AppTheme` (so it carries no app-specific theme; `Notice` uses `AppTheme.Tone`). `Mire.Demo.Agent` now renders its transcript through `Mire.Agent.ChatTranscript`, passing an `AppTheme` assembled from its own styles — verified byte-identical via `--dump`. (`ToolCallView`, `ThinkingBlock`, `DiffView` (unified), `FileTree`, and `TaskTimeline` ship as `TranscriptBlock` variants rendered by `ChatTranscript`, not standalone widgets.)
  - **`PromptBox`** — the demo's `PromptInput.fs` (a `TextBuffer`/`TextEdit` wrapper + block-cursor render) moved into `Mire.Agent` under the SPEC name; the demo's prompt renders through it.
  - **`ApprovalModal`** — `ApprovalModal.view` (title/intro/command/risk + Accept/Deny) + `buttonHit` (click hit-test), styled by `AppTheme`. The demo's permission modal renders and click-activates through it; the accept/deny behavior stays app-side. Verified via `--dump` + a pty click on Accept firing its toast.
  - **`samples/AgentShell`** — a minimal coding-agent shell (SPEC's headline example) composing `ChatTranscript` + `PromptBox` + `ApprovalModal` on `AppTheme.defaultTheme` with **zero app-specific theme code** — the proof the agent layer composes into a real app. Type to chat; `run` opens a tool-approval modal. Verified via `--dump` (both states) and an end-to-end pty run (type → submit → `run` → approve).
  - **`DiffView`** — `DiffView.render` shows a `DiffHunk` list in `Unified` or `Split` (side-by-side before/after) mode with per-hunk accept/reject markers (`HunkStatus`) and a selection highlight; pure, with the app owning the hunks/selection/status (MVU). `splitColumns`/`statusMark` are unit-tested; the `agentShell` sample's `diff` command drives it interactively (j/k select, a/r accept/reject, s toggles mode). Verified via `--dump` + a pty run (pressing `a` accepts the selected hunk: 0 → 1 checkmark).
  - Remaining 0.5.0 work: folding completion/history into `PromptBox` and virtualization/follow-tail into `ChatTranscript`, and the optional widget-gallery app (see `ROADMAP.md`).

## [0.4.0] - 2026-06-17

First deliberate release: the core framework — runtime, layout engine, and the
full base-widget layer — targeting modern Kitty-compatible terminals. The
agent-domain layer (`Mire.Agent`) is still prototyped only in `Mire.Demo.Agent`
and lands in 0.5.0.

### Added

- **OSC 8 hyperlinks (`Style.WithLink`).** `Style` gains a `Link: string option` field; `Style.WithLink url` attaches an OSC 8 target to a run. The `Diff` writer brackets a linked run in OSC 8 open/close (tracking link state across cursor moves and closing any open link at frame end), and `ToAnsi` ignores the link (it's not an SGR attribute). Because `Link` is part of the struct, structural equality splits diff runs at link boundaries automatically. `Widgets.Markdown` now carries each `[text](url)` link's real URL onto the styled link text. Verified end-to-end through a pty: the Agent demo's `markdown` sample emits exactly one `OSC 8 open`/`close` pair around the link.
- **OSC 52 clipboard (`Cmd.setClipboard`).** A new `Cmd.setClipboard text` copies to the system clipboard; the runtime interprets it like `Cmd.quit` — writing the OSC 52 sequence straight to the terminal (outside the cell diff, since it paints nothing).
- **Kitty key event types (press / repeat / release).** The runtime now pushes `CSI > 3 u` (disambiguate **+ report event types**), and `InputParser` decodes the `:event` sub-parameter of the `CSI u` modifier field into the existing `Press`/`Repeat`/`Release`. To keep the common case unchanged, the runtime **drops `Release` events by default** (else every keystroke would fire twice); `Program.withKeyReleases true` opts in. `Repeat` always passes through (held keys still work).
- **Bracketed-paste reassembly across reads.** A paste split across multiple `read()`s used to arrive as several `Paste` events (or garbage). The runtime now carries an unfinished paste (no end marker yet) and prepends it to the next read, reassembling it into one `Paste` — bounded at 1 MiB so a lost end marker can't grow the buffer without limit. The decision is a pure, tested function (`InputParser.stepPasteBuffer`).

### Changed

- **`Mire.Demo.Agent` adopts framework facilities (first wave).** Two new built-in commands dogfood the protocol work: `copy` puts the last assistant message on the system clipboard via `Cmd.setClipboard` (OSC 52, with a confirmation toast), and `quit`/`exit` leaves cleanly via `Cmd.quit` (Ctrl+C still works). Both route through `applyResponse`, so they work from the prompt *and* the command palette, and both appear in the palette/`help` list. `Blocks.spinner` now delegates to the framework `Spinner.frameOf`/`Spinner.braille` instead of a hand-rolled frame table, and the transcript pane renders through `ScrollView.vertical` (track/thumb scrollbar) instead of a bare `Scroll`. Transcript markdown links already carry OSC 8 hyperlinks for free via the framework's `Markdown` → `Style.Link` wiring. Verified through a pty: typing `copy` emits one OSC 52 write whose payload decodes to the rendered message, and `quit` exits without Ctrl+C.
- **`Mire.Demo.Agent` adopts framework facilities (second wave — the structural migrations).** The `Ctrl+P` palette now ranks with the fuzzy `CommandPalette.filter` (best-first subsequence) instead of a substring match; the transcript gets a `ScrollView` scrollbar; overlays center via `Overlay.centered`; the permission modal, skill explorer, and the three `/mcp` views render through `Widgets.Modal.modal` (title slot + backdrop); the `TableBlock` card renders through `Widgets.Table.view`; the mouse wheel scrolls the transcript (SGR 1006) and a left-click activates the permission modal's Accept/Deny buttons (a hand-mirrored hit-test, since retained region hit-testing is ROADMAP v0.5); and the base prompt ⟷ transcript Tab ring uses `Mire.Layout.Focus` (replacing a bespoke `FocusRegion` DU). Overlay routing stays a `Model.Overlay` modal `match` by design — the overlays are a single-at-a-time stack carrying their own state, so focus-trapping would be redundant (unlike `Mire.Demo.Feed`'s genuine two-pane base ring). Each migration verified via `--dump` and/or pty capture; see `DEMO-TODOS.md`.
- **The brand is the framework default theme.** `brand/palette.fs` (`Mire.Brand.Palette`) is now compiled into the `Mire` assembly as the single source of truth (demos stopped compiling their own copy). The predefined `Style.*` primitives, `AppTheme.defaultTheme`, and `Markdown.defaultStyle` are all rebuilt from the palette — emerald accent, neutral hierarchy, inverse-video selection, light-emerald markdown headings — so any app gets the on-brand look with no theme code. Added `AppTheme.Tone` + `AppTheme.toneStyle` (the success/warning/danger/info/neutral mapping the demos hand-rolled). The F# palette's emerald was also refreshed to `#1A9A7E` to match the web palette (`palette.css`/`palette.json`), which had drifted ahead of it. Verified via pty that the demos render the refreshed emerald (`38;2;26;154;126`) and not the stale `#1A8870`.
- **Removed dead scaffolding in `Mire.Protocol`.** Deleted the unused `tcgetattr`/`tcsetattr`/`ioctl` libc externs (raw mode uses the `stty` subprocess; size uses `Console.WindowWidth/Height`) and the stale "For now, use Console APIs" comment in `setupRawMode`.

### Removed

- **Consolidated to three demos: `Mire.Demo.Agent`, `Mire.Demo.Feed`, `Mire.Demo.Spreadsheet`.** Deleted `Mire.Demo.List`, `Mire.Demo.KitchenSink`, and `Mire.Demo.Minesweeper`. The Agent demo is now the canonical/comprehensive showcase — its theme, styling, and behavior are being promoted to the framework's core approach (subsequent entries). The KitchenSink gallery's widget coverage will return as a dedicated gallery app once the brand-default theme and Agent refactor land (tracked in `ROADMAP.md`). The solution dropped from eight projects to five (framework + 3 demos + tests); `Mire.slnx`, the `Mire.Tests` Minesweeper `Board` tests + project reference, the `justfile` recipes, and the docs follow.

### Fixed

- **Wide glyphs no longer leave a ghost right-half.** A wide (width-2) glyph used to write only its left cell and leave the trailing column untouched, so replacing it with narrower content across frames left the glyph's stale right half on screen. The glyph now records `Width = 2` and writes a *continuation* cell (empty grapheme) at the trailing column — distinct from a blank, so the diff repaints that column when the glyph is replaced. The continuation absorbs into the glyph's diff run (no stray output), and a combining mark following a wide glyph now steps back over the continuation to attach to the base glyph. Fixed in both `Surface.Write` and `Surface.WriteClipped`; 4 tests.
- **Spreadsheet (and Feed) opened in a fixed-size window regardless of the terminal.** Both demos hardcoded `Size.Create(100, 30)` at `init`, and the runtime only emits a `TerminalResize` once the size *changes* from startup — so on any terminal that wasn't exactly 100×30 the grid stayed boxed into that window until the user physically resized. `init` now seeds the model from `TerminalMode.getTerminalSize ()` (the pattern the Agent demo already used), so the first frame fills the real terminal. Verified through a pty: a 160-column terminal now paints out to column 160, an 80-column one to column 80.

- **Full-width selection highlight in `Mire.Demo.Agent`'s other lists.** The command palette got the `Backdrop.behind` full-row treatment earlier, but its lookalikes didn't: the slash/`@mention` completion popup, the skill-explorer list, and the `/mcp` server + action lists still shaded only the cells under the glyphs. The plain-string lists (skills, MCP actions) now use the framework's `ListView.row` (which does this correctly); the multi-column rows (completion, MCP servers) apply the same backdrop pattern as the palette, with the whole selected row switching to the selection style. Verified against real runtime output through a pty: the selected row renders as one continuous run with the selection background spanning border to border.

### Changed

- **Demo projects renamed to the `Mire.Demo.*` family.** `Mire.Demo` → `Mire.Demo.List`, `Mire.AgentDemo` → `Mire.Demo.Agent`, `Mire.FeedDemo` → `Mire.Demo.Feed`, `Mire.SpreadsheetDemo` → `Mire.Demo.Spreadsheet`, `Mire.MinesweeperDemo` → `Mire.Demo.Minesweeper` — one shared prefix that groups + sorts together. Dirs, `.fsproj`s, `namespace`/`open` decls, `Mire.slnx`, the `Mire.Tests` references, the `justfile`, and the living docs all follow; the dated design specs keep their historical names. No behavior change.

### Added

- **`Mire.Demo.KitchenSink` — a comprehensive widget showcase.** A sidebar + detail-pane gallery covering all 20 widget categories (Text, Styles, Box/Panel, Separator/Badge/KeyHint, Spinner, ProgressBar/Gauge, Tabs, Toggle, ListView, Table, ScrollView, Input, TextArea, Modal, Toast, Completion, Tooltip, Positioned, SplitView, Markdown), each in several configs/states. Spinners and the animated bar run off a `Sub.Every` tick; the data/editing entries are driveable when focused (Tab into the pane); the app's own sidebar|detail layout dogfoods `SplitView`. A single `Detail.render` backs both the live view and a headless `--dump` (48×12 per entry — the golden-frame source).
- **`Spinner` / `ProgressBar` / `Tabs` / `Toggle` widgets (`Mire.Widgets`).** Four small base widgets the demos hand-roll today (pure render, app owns state). `Spinner.frameOf` maps an app-owned tick to a braille frame (wraps; negative/empty safe) + `view`/`labeled`. `ProgressBar.view` is a `█`/`░` determinate bar for a 0..1 fraction; `.gauge` overlays the centered percentage. `Tabs.strip` is a horizontal tab strip with the active tab styled. `Toggle.checkbox`/`radio`/`switch` are the selection-control glyphs. 8 tests.
- **`Markdown` widget (`Mire.Widgets.Markdown`).** The AgentDemo's line-oriented renderer promoted into the framework, parameterized by a `MarkdownStyle` record (headings, quote, link, code-token colors, rule) instead of hardcoded theme refs; inline bold/italic/strike derive from the base style, and `Mention` is optional. Ships a neutral `defaultStyle`. `render`/`wrap`/`wrapLines` take the style set first; the AgentDemo is migrated onto it (its local `Markdown.fs` deleted, `Theme.markdown` threaded through). 4 tests.
- **Golden-frame snapshot tests.** A `gridText` helper renders a node and returns the whole `w×h` cell grid as text; the `GoldenFrame` suite asserts exact full grids for a `Box` and a multi-widget dashboard (`Tabs` + `ProgressBar` + `Toggle`), plus determinism and a row-width invariant. (Ticks the ROADMAP "promote `--dump` scenarios into golden-frame tests" item.)
- **`SplitView` + `Tooltip` widgets (`Mire.Widgets`).** Two small base widgets over existing primitives, pure render (no input). `SplitView.split direction firstSize dividerStyle first second` (+ `horizontal`/`vertical`) lays two panes along an axis with a 1-cell `Filled` divider gutter between them; the app owns the split position (`Cells n` for a resizable/app-driven position, `Fraction f` for a proportional one) and the second pane fills the remainder — the convenience the demos hand-roll today as raw `Dock`/`Stack` fractions. The gutter auto-spans the cross axis, so it shows with a background-bearing `dividerStyle` (a `Style.Default` gutter is an invisible 1-cell gap). `Tooltip.view areaW areaH anchorX anchorY width borderStyle textStyle lines` is an anchored bordered doc popup on `Overlay.atPoint`: it sits just below the anchor, flips above when it would run off the bottom, and is clamped on-screen (mirroring `Completion`'s placement); the caller wraps `lines` to fit `width - 2`. 4 tests.
- **`TextArea` (multi-line) + a reusable editing keymap — closes v0.3.** Three layers land together, keeping the framework's "actions, not bindings" discipline:
  - **`Mire.Core.TextBuffer`** grows from single-line to `\n`-aware: `wordLeft`/`wordRight`, `deleteWordBack` (rewritten on `wordLeft`) / `deleteWordForward`, `lineStart`/`lineEnd`, `up`/`down` (column-clamped to the target line, **no sticky desired-column**, no-op at the first/last line), and `cursorRowCol`. Word boundaries are whitespace-aware **including `\n`**, so word ops never silently run across a line break.
  - **`Mire.Core.TextEdit`** (new file) is the reusable keymap, as data: an `EditAction` DU (`InsertText`/`Newline`/`DeleteBack`/`DeleteForward`/`DeleteWordBack`/`DeleteWordForward`/cursor + word + line + doc moves), `apply` (action → buffer), an **overridable** `defaultKeymap : KeyEvent -> EditAction option` (conventions only — typing, Backspace/Delete, Ctrl/Cmd/Alt word chords, arrows, Home/End; returns `None` for keys it doesn't own so the app/quit can claim them), and `applyInput`/`applyInputWith` which also route a `Paste s` event into an insert. Apps wire keys → actions by using the defaults, wrapping them to pre-empt a few keys, or building a keymap from scratch — the framework never hardcodes a binding into the buffer or the runtime.
  - **`Mire.Widgets.TextArea.render width height textStyle cursorStyle focused buf`** — a multi-line block-cursor view over the buffer: vertical + horizontal scroll-to-cursor (no soft-wrap), reusing `Input`'s left/at-cursor/right cursor split on the caret row and `ScrollView.clampOffset` for the vertical window.
  - **Configurable quit (`Program.withQuitOn`).** The runtime's hardcoded Ctrl+C intercept became a default-but-overridable policy `QuitOn : InputEvent -> bool` (default = Ctrl+C, behavior unchanged), consulted before `MapInput`. `withQuitOn (fun _ -> false)` makes Ctrl+C an ordinary key and exits only via `Cmd.quit`. Doc comments that baked key names into Core (`TextBuffer.deleteWordBack`, `Focus.next`/`prev`) now name the _action_ instead.
  - **Dogfood:** `Mire.AgentDemo`'s `PromptInput` placeholder is replaced by the real stack — it wraps a `TextBuffer`, renders via `TextArea` (a real caret), and feeds input through `TextEdit.applyInput`, so typing, `Backspace`/`Delete`, Ctrl/Cmd+Backspace word-delete, and ←/→ all work. Real bracketed-`Paste` events are now handled (a big/multi-line paste still collapses to a `[Pasted #n]` chip — driven by an actual `Paste`, not the old `c.Length >= 16` heuristic). The command palette reads its query from the same editing event. 0 warnings; new `TextBuffer`/`TextEdit`/`TextArea`/`QuitOn` tests (128 total, all green).
- **`Separator` / `Badge` / `KeyHint` widgets (`Mire.Widgets`).** The small status-bar/panel trio: `Separator.horizontal width` (`─`) / `Separator.vertical height` (`│`) rules; `Badge.badge style label` — a padded, toned pill (the caller supplies the tone via `style`); `KeyHint.hint keyStyle labelStyle key label` — a styled key glyph + label chip (e.g. `Ctrl+P palette`). 3 tests.
- **`Completion` widget + `Overlay.atPoint` (`Mire.Widgets`).** `Overlay.atPoint x y w h areaW areaH child` places a sized child at a caller-supplied point, clamped to stay fully on-screen (the cursor/anchor-popup primitive — the app already knows the caret position and screen size, so it's a `Dock`-spacer helper, not a layout-node change). `Completion.view` builds on it: a small bordered, selectable list anchored just below the caret; the app filters candidates with `CommandPalette.matches`/`filter` and tracks the selection. 2 tests. (Above/below flip when low on space is a follow-up.)
- **`CommandPalette` widget (`Mire.Widgets.CommandPalette`).** A centered, fuzzy-filtered command surface. The pure core is `matches` (case-insensitive subsequence — `"tp"` matches `"ToolPanel"`; empty query matches all) and `filter` (keep matching items, order preserved) — both reusable by a future `Completion`. `view` composes a `Modal` (title + `❯ query` line) over a selectable `ListView` of the already-filtered items; the app owns the query + selection and pairs open/close with `Focus.pushTrap`/`popTrap`. 3 tests. (`Mire.AgentDemo`'s hand-rolled palette is a follow-up adopter.)
- **`Table` widget (`Mire.Widgets.Table`).** A sticky-header table over `Length`-width columns. A `Column<'row,'msg>` carries a `Header`, a `Width`, and a `Render : 'row -> LayoutNode` cell renderer (so cells can be styled/custom); `Table.textColumn` is a plain styled-text convenience. `Table.view height headerStyle selStyle topRow selected columns rows` draws the header above a `height`-row window of the data starting at `topRow`, with a full-width highlight on the `selected` row — the app owns the offset + selection (like `ListView`/`ScrollView`, and composes with the `ScrollView` offset helpers). The grid demos (`Mire.SpreadsheetDemo`, `Mire.MinesweeperDemo`) motivate it; a demo migration is a follow-up. 1 test.
- **Mouse, bracketed paste, and focus input decoding (`Mire.Protocol.InputParser`).** The parser produced only `Key` events before; the `Mouse` / `Paste` / `FocusGained` / `FocusLost` `InputEvent` cases existed but were never emitted. Now `parseBytes` decodes: **SGR 1006 mouse** (`ESC [ < b ; x ; y` then `M`/`m`) → `Mouse` with 0-based coords, button (incl. scroll wheel), and Shift/Alt/Ctrl modifiers; **bracketed paste** (`ESC [ 200 ~ … ESC [ 201 ~`) → `Paste` of the inner text; **focus** (`ESC [ I` / `ESC [ O`) → `FocusGained` / `FocusLost`. `Runtime.run` now also enables bracketed paste (`?2004h`), which it wasn't writing before. The non-Key decoders run ahead of the Key decoder, so all existing keyboard decoding is unchanged. 6 tests. (Apps opt in by matching these cases in `mapInput`.)
- **`ScrollView` widget (`Mire.Widgets.ScrollView`).** A vertically-scrolling viewport with a track/thumb scrollbar, on the existing `Scroll` node. `ScrollView.vertical viewportH contentH offset trackStyle thumbStyle content` renders the scrolled content beside a 1-cell scrollbar whose thumb is sized and positioned from the viewport/content heights (just the track when content fits). The app owns the offset (like `Scroll.vertical`); the pure helpers `clampOffset` / `toBottom` / `atBottom` make jump-to-bottom, follow-tail, and paging one-liners in `update`. `Mire.FeedDemo`'s reader pane now uses it (its hand-rolled `Scroll.vertical` + right-margin gains a real scrollbar). 2 tests.
- **`Toast` widget (`Mire.Widgets.Toast`).** The layout half of auto-dismissing notifications, on the `Positioned` node: `Toast.stack placement width cardHeight cards` places a column of toast cards at a `Placement` (e.g. `TopRight`), each `cardHeight` tall with a blank row between; `Toast.card border title body` is a default bordered card. The app owns the toast list and expires entries with a `Sub` timer. `Mire.AgentDemo`'s `toastLayer` now uses it, replacing its hand-rolled `Dock`-spacer top-right placement. 1 test.
- **Focus manager core — `Mire.Layout.Focus`.** A pure, total focus value type keyed on `Core.RegionId`: a base `FocusRing` (tab order + current) plus a stack of modal trap rings. `ofOrder` / `current` / `isFocused` / `focus` / `next` / `prev` / `pushTrap` / `popTrap` / `isTrapped` — every navigation and query acts on the innermost ring, so a modal trap is automatic (base ids are unreachable while trapped) and `popTrap` restores focus exactly where it was left. ### Fixed

- **Growing the terminal left the newly-exposed rows/columns unpainted.** `Runtime.run` kept the previous `Surface` on resize, so `Diff.compute` only diffed the `min(old, new)` overlap. The resize branch now clears `PreviousSurface`, forcing a from-scratch full repaint at the new size (fixes stale cells on both grow and shrink). Two `Diff` characterization tests pin the overlap-vs-full-repaint premise; the runtime branch itself needs a tty to exercise.
- **`Box.panel` / `StatusBar.statusBar` overlapped their children.** A `Box` renders a single child filling its inner rect; these helpers passed multiple children, which overdrew (only the last survived). They now flow their children through an explicit vertical / horizontal `Stack`. `Box` stays a single-child container by design — nest a `Stack` for multiple. 2 new tests.
- **Feed filter reset to "show all" when reopened** (`Mire.FeedDemo`) — with a subset filter active, pressing `f` rebuilt the picker with every feed checked, so an unchanged Enter collapsed the selection back to all feeds. The picker now seeds `Checked` from the active `Selected` set (falling back to all feeds only in the unfiltered state), so reopening preserves the current subset.
- **Spreadsheet active cell could scroll off-screen horizontally** (`Mire.SpreadsheetDemo`) — after the grid grew to 26 columns, terminals showing only the first few columns lost the cursor once it moved past the rendered range, because the model tracked only vertical scroll. Added a `LeftCol` offset + `keepColInView` (mirroring the row path), so the rendered column window follows the cursor (and the reference-picker target).
- **Selection highlight only covered the text, not the full row** (command palette; any styled-background row). A styled `Text` node paints a background only under its glyphs, leaving the gaps and the row's tail at the default colour. Selected rows now wrap their content in `Backdrop.behind`, filling the whole row with the selection colour.
- **Arrow keys didn't work in application-cursor-key mode (DECCKM)** — notably in JetBrains' JediTerm, which sends `ESC O A`/`B`/`C`/`D` for the arrows where most terminals send `ESC [ A`/…. The SS3 (`ESC O …`) branch only decoded F1–F4/Home/End, so those arrows were dropped (they worked in Kitty/Ghostty, which use the `ESC [` form). Added `ESC O A/B/C/D` → arrow keys. 3 new tests; verified end-to-end (SS3 Down scrolls `Mire.Demo`).
- **Modifier keys (Ctrl/Alt/Shift/Super) were never detected** in Kitty-protocol terminals (Ghostty, Kitty) — e.g. `Ctrl+P` / `Ctrl+O` did nothing while plain keys worked. `Runtime.run` enables the Kitty keyboard protocol (`CSI > 1 u`), so those terminals send modified keys in the Kitty **`CSI u`** form (`Ctrl+P` → `ESC [ 112 ; 5 u`), but `InputParser` only decoded legacy sequences and dropped `CSI u`. (Diagnosed by capturing raw bytes from Kitty, Ghostty, and JediTerm — JediTerm ignores the protocol and still sends legacy `0x10`, which already worked.) Fixed by rewriting the CSI parser to decode the Kitty encoding — `ESC [ <codepoint> ; <modifiers> u` → `Char`/named key + `KeyModifiers` (Kitty _super_ → `Meta`) — plus modified navigation keys (`ESC [ 1 ; <mod> A/B/C/D/H/F`, `ESC [ <n> ; <mod> ~`). All legacy decoding (plain arrows, `Ctrl+letter`, `Shift+Tab`, F-keys) is preserved. 7 new `InputParser` tests; verified end-to-end (real Kitty `Ctrl+P` opens the palette, `Ctrl+O` the skill explorer in `Mire.AgentDemo`). Release/repeat event types are still not requested (only the disambiguate flag is pushed).
- **No keyboard input was detected at all** — every demo looked frozen (no key, arrow, or scroll registered). `TerminalMode.stdinAvailable()` called libc `poll` with the `pollfd` passed as a managed **array** (`poll([|pfd|], …)`); the kernel's write to `revents` was never marshaled back to managed memory, so `pfd.Revents &&& POLLIN` was always `0`, `stdinAvailable` always returned `false`, and `InputParser.readEvent` always returned `None`. Fixed by passing the struct **by reference** — `extern int poll(PollFd& fds, …)` called as `poll(&pfd, …)` — which marshals as `pollfd*` (exactly what poll wants for `nfds = 1`) and is in/out, so `revents` is written back. Verified end-to-end on macOS (.NET 10): arrows, characters, and Home/End now drive `Mire.Demo`. (`read`'s `byte[]` buffer was unaffected — primitive arrays are pinned in place, so its writeback always worked.)
- `Console.KeyAvailable` `InvalidOperationException` in raw mode — the App runtime now works end-to-end
- `Surface` constructor no longer crashes on invalid terminal sizes (0×N or negative dimensions); clamps to minimum 1×1
- `TerminalMode.getTerminalSize()` now returns `None` instead of invalid `Size` when `Console.WindowWidth/Height` report zero or negative
- `Mire.App` `AsyncCmd` dispatch now correctly passes the `Async<unit>` to `Async.Start` without wrapping in a redundant `async { do ... }` block
- `Mire.Layout` incomplete pattern match warning on `DockPosition` — added catch-all cases for `Content`/`Fill` lengths inside `Top`/`Bottom`/`Left`/`Right`
