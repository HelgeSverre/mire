`Mire.Widgets` is a library of pure view helpers. Each returns a `LayoutNode<'msg>`,
holds no state, and is styled by the values you pass it — so the same widget renders any
theme, and your model owns the state. The screenshots below are **real output**: each is
the actual widget laid out through `Layout.measure`/`render` on the default brand theme,
serialized to SVG (generated by the `Mire.Shots` tool). Run them yourself with
`just gallery`.

Editing widgets are covered in [Edit text](/docs/how-to/edit-text/); the agent widgets
in [the agent layer](/docs/reference/agent-layer/). For how to compose overlays, see
[Show overlays and modals](/docs/how-to/overlays-and-modals/).

## Not pictured

A few widgets are layout glue or anchored popups that don't screenshot meaningfully on
their own:

- `Separator.horizontal width style` / `Separator.vertical height style` — a `─` or `│` rule.
- `Backdrop.solid style` / `Backdrop.behind style child` — a solid fill / a full-width row highlight (lists and tables use it for selection).
- `SplitView.horizontal firstSize dividerStyle left right` / `SplitView.vertical …` — two panes with a divider.
- `ScrollView.vertical viewportH contentH offset trackStyle thumbStyle content` — a viewport with a scrollbar, plus `toBottom` / `clampOffset` / `atBottom` offset helpers.
- `Tooltip.view areaW areaH anchorX anchorY width borderStyle textStyle lines` — an anchored doc popup.
- `CommandPalette.view …` — a fuzzy command surface (Modal + ListView); `CommandPalette.filter query candidates` is the reusable ranker.
- `Overlay.centered` / `Overlay.positioned` / `Overlay.atPoint` — place a sized layer.

See the [layout reference](/docs/reference/layout/) for the nodes these build on.
