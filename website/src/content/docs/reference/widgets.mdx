`Mire.Widgets` is a library of pure view helpers. Each returns a `LayoutNode<'msg>`,
holds no state, and is styled by the values you pass it — so the same widget renders
any theme, and your model owns the state. The previews below are **real output**: each
is the actual widget laid out through `Layout.measure`/`render` on the default theme,
serialized to SVG (by the `Mire.Shots` tool). Open a widget for its signature, a
parameter table, and an example. Run them live with `just gallery`.

## Not pictured

A few widgets are layout glue or anchored popups that don't preview meaningfully on
their own:

- `Separator.horizontal width style` / `Separator.vertical height style` — a `─` or `│` rule.
- `Backdrop.solid style` / `Backdrop.behind style child` — a solid fill / a full-width row highlight.
- `SplitView.horizontal firstSize dividerStyle left right` / `SplitView.vertical …` — two panes with a divider.
- `ScrollView.vertical viewportH contentH offset trackStyle thumbStyle content` — a viewport with a scrollbar, plus `toBottom` / `clampOffset` / `atBottom` helpers.
- `Tooltip.view areaW areaH anchorX anchorY width borderStyle textStyle lines` — an anchored doc popup.
- `CommandPalette.view …` — a fuzzy command surface; `CommandPalette.filter query candidates` is the reusable ranker.
- `Overlay.centered` / `Overlay.positioned` / `Overlay.atPoint` — place a sized layer.

See the [layout reference](/docs/reference/layout/) for the nodes these build on.
